Overview

Sidekar is a local-first tool. Your data stays on your machine.

CLI & MCP Server

The sidekar binary runs entirely on your local machine. It does not phone home, collect telemetry, or transmit any data to external servers. Browser sessions, desktop automation, and inter-agent communication all happen locally.

Chrome Extension

The Sidekar Chrome extension connects exclusively to 127.0.0.1 (localhost) via WebSocket. It does not:

  • Send any data to remote servers
  • Collect analytics or telemetry
  • Track browsing history or behavior
  • Store or transmit page content outside your machine

The extension reads page content (DOM text, accessibility tree, screenshots) only when the local sidekar process sends a command. A shared secret authenticates the connection between the extension and the local bridge.

The extension requests <all_urls> host permission because AI agents may need to interact with any page you direct them to. This permission is never used to access pages without an explicit command from your local agent.

Website

The sidekar.dev website is a static site. It uses no cookies, analytics, or tracking scripts. The optional login flow (for license validation) uses a device-auth code exchange and stores a token locally in ~/.config/sidekar/.

Data Storage

All sidekar data is stored locally:

  • Chrome profiles: ~/.sidekar/profiles/
  • Configuration: ~/.config/sidekar/sidekar.json
  • Extension bridge secret: ~/.sidekar/ext-secret (0600 permissions)
  • Temporary files: /tmp/sidekar-*

Third-Party Services

Sidekar does not integrate with any third-party analytics, advertising, or data-collection services. The sidekar update command checks GitHub Releases for new versions — this is the only outbound network request the CLI makes.
