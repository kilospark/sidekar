The sidecar for
AI agents

Sidekar is an automation sidecar for AI CLI agents like Claude Code, Codex, Cursor, Copilot, and Gemini CLI. It adds a local message bus with optional cross-machine relay, token-efficient browser automation through a dedicated CLI and optional Chrome extension, macOS desktop automation, background monitoring and cron, and encrypted key-value storage with TOTP—so agents can finish logins and act on your behalf without exposing secrets in chat. Optionally sign in to sidekar.dev for a web terminal and dashboard with remote session visibility. One Rust binary. Runs alongside your existing agent; it doesn't replace it.

Why sidekar

CLI agents can reason and write code, but they can't see a browser, talk to each other, or log in to anything. sidekar fills every gap: a message bus that works across machines, a real browser they can see and control, macOS desktop access, encrypted credentials with TOTP for 2FA, and background scheduling. The pieces compose: an agent on your laptop can hand off to one on a server, which logs in to a service using stored credentials, runs a workflow, and sends the results back.

What it does

Agent communication bus

Agents find each other and exchange typed messages -- requests, responses, handoffs -- whether they're in the same terminal or on different continents. A local SQLite broker handles same-machine delivery; a WSS relay tunnel carries everything else.

Web terminal

Start an agent on your desktop, then watch it work from your phone. Every PTY session streams live to sidekar.dev/terminal with full xterm rendering and scrollback. No SSH, no port forwarding -- just open a browser anywhere.

Browser automation

Raw Chrome DevTools Protocol over WebSocket, plus an optional Chrome extension for your everyday profile. Navigate, click, type, read pages, screenshot, capture network traffic -- all in your actual Chrome with your cookies and logins.

Desktop automation

macOS Accessibility API integration. Find and interact with native app elements, take screenshots, launch and manage applications. Covers UI that lives outside the browser.

Background automation

Monitor open tabs for meaningful changes and run commands on a cron schedule. Results flow back through the bus, so agents pick them up automatically. Keeps working after you close the lid.

Credentials and encrypted storage

The missing piece for fully autonomous agents. An encrypted KV store holds usernames, passwords, and API keys (AES-256-GCM at rest). TOTP generates 2FA codes on demand. Combined with browser automation, agents can log in to any service end-to-end without secrets ever appearing in chat or tool output.

Getting started

01

Install

curl -fsSL https://sidekar.dev/install | sh downloads the binary, updates your PATH, and runs sidekar install so the skill is available in supported agents.

02

Chrome extension (optional)

Clone or download the sidekar repo and load the extension/ directory as an unpacked extension. Start the bridge with sidekar ext-server or any sidekar ext … command, paste the shared secret in the toolbar popup, and connect to automate your normal Chrome profile.

03

sidekar login (optional)

Authenticate with sidekar.dev to unlock the relay tunnel, the web terminal, and the dashboard. This is what connects agents and sessions across machines.

04

Launch your agent

Run sidekar <agent> [args…] for any agent CLI on your PATH (for example sidekar claude, sidekar codex). Sidekar wraps the PTY, registers the bus, and connects browser tooling.

What you can ask

What agents can do with sidekar.

Coding
"Build this component, then open the dev server and verify it renders correctly."
Launches Chrome, navigates to localhost, screenshots the result, catches visual bugs before you review.
Research
"Research how Stripe handles idempotency keys and summarize the best practices."
Searches the web, reads multiple pages in parallel, synthesizes findings from real documentation.
Testing
"Fill out the signup form with test data and make sure the validation errors show up."
Navigates to the form, fills fields, submits, reads error messages, reports what passed and what didn't.
Multi-agent
"Build the API, then tell the frontend agent it's ready to integrate."
Finishes the work, discovers the other agent on the bus, sends a handoff with context. The frontend agent picks it up and starts testing.
Remote
"Ask the agent on the staging server to run the migration and report back."
Finds the remote agent through the relay tunnel, sends a request, waits for the response with results.
Monitoring
"Watch the Grafana dashboard and let me know if the error rate spikes."
Opens the dashboard, monitors tab title changes, alerts you when something triggers.
Desktop
"Check if there are any unread messages in Slack."
Uses the macOS Accessibility API to inspect Slack's UI, finds unread indicators, reports back.
Credentials
"Log in to the AWS console -- the credentials and TOTP secret are in the KV store."
Pulls the password from encrypted storage, generates the current 2FA code, fills the login form, and navigates to the dashboard.
